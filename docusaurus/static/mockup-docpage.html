Quick Start Guide

Last updated: March 2026 · 8 min read
💡
Tip
Strapi 5 requires Node.js 18.15+ and npm 6+. We recommend using the latest LTS version.

Strapi offers a lot of flexibility. Whether you want to go fast and quickly see the final result, or prefer to dive deeper into the product, we got you covered. For this tutorial, we'll go for the quick option.

Step 1: Create a new project

Create a new Strapi project using the CLI. The following command will create a new project with the default SQLite database:

terminal 
npx create-strapi@latest my-project --quickstart

This will create a new Strapi project in a my-project directory, install all dependencies, and start the development server.

Step 2: Register the first administrator

After creating your project, the admin panel opens in your browser at http://localhost:1337/admin. Complete the form to create the first administrator account.

ℹ️
Note
The first user created is the Super Admin, who has access to all features and settings.

Step 3: Create content types

Navigate to the Content-Type Builder in the admin panel. This is where you'll define the structure of your content. Think of content types as the data models for your application.

Creating a collection type

Collection types are content types that can have multiple entries. For example, a "Restaurant" collection type could have many restaurant entries. Click Create new collection type and give it a display name.

config/api/restaurant/content-types/restaurant/schema.json 
{
  "kind": "collectionType",
  "collectionName": "restaurants",
  "info": {
    "singularName": "restaurant",
    "pluralName": "restaurants",
    "displayName": "Restaurant"
  },
  "attributes": {
    "name": { "type": "string" },
    "description": { "type": "richtext" }
  }
}

Adding fields

Strapi provides a wide range of field types to model your content. The most common fields include text, rich text, media, relations, and components. Each field can be configured with validations and custom settings.
