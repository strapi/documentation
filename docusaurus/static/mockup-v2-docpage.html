Quick Start

Get a full-stack Strapi application running locally in under 5 minutes.

💡
Prerequisites
Node.js ≥ 18.15 and npm ≥ 6. We recommend the latest LTS release.

Create your project

Scaffold a new Strapi application with a single command. The --quickstart flag uses SQLite so you don't need to configure a database.

terminal 
$ npx create-strapi@latest my-project --quickstart

This installs dependencies and starts the dev server on localhost:1337. The admin panel opens automatically in your browser.

ℹ️
First user is Super Admin
The first account you create has full access to every feature and setting in Strapi.

Define your content

Open the Content-Type Builder from the left menu. Create a new Collection Type — for example, Restaurant.

schema.json 
{
  "kind": "collectionType",
  "info": {
    "singularName": "restaurant",
    "pluralName": "restaurants",
    "displayName": "Restaurant"
  },
  "attributes": {
    "name":        { "type": "string" },
    "description": { "type": "richtext" },
    "rating":      { "type": "integer" }
  }
}

Query your API

Strapi auto-generates REST and GraphQL endpoints for every content type. No configuration needed — just publish an entry and fetch it.

⚠️
Permissions
By default, APIs are locked down. Go to Settings → Roles → Public and enable find / findOne for your content type.
terminal 
$ curl http://localhost:1337/api/restaurants

{
  "data": [
    {
      "id": 1,
      "name": "Biscotte",
      "description": "A cozy place.",
      "rating": 5
    }
  ],
  "meta": { "pagination": { "total": 1 } }
}

What's next?

  1. Explore the admin panel
    Create content entries, set up roles and permissions, manage media files.
  2. Connect a frontend
    Use the REST or GraphQL API with Next.js, Nuxt, Gatsby, or any framework.
  3. Deploy to Strapi Cloud
    One command to push your project to production with managed hosting.
